**L-35.** Cross-model adherence is NOT guaranteed by exposing the MCP tools — a non-Claude model may bypass the tool surface entirely and reach for its own built-ins, the CLI, and raw file edits. Cursor's Composer 2.5 Fast, with the cz_* MCP tools live, made 0 MCP calls across 142 tool calls driving a full gameplan: it recovered from a PATH miss by discovering the portable `uvx … clauderize ops` JSON-batch path on its own (validating that a CLI-reachable fallback for every tracked write [L-05] is load-bearing for cross-MODEL, not just no-MCP sessions), but ALSO hand-edited tracked docs — the frontmatter/graph-corruption anti-pattern the blessed writes exist to prevent. Three takeaways: the CLI/ops fallback earns its place; trust-but-VERIFY a non-Claude model's self-reported close-out against engine state (its plausible summary claimed graph entities a `reindex` showed were never created); and the 'never hand-edit' floor instruction alone does not steer every model — a guard that detects unmodeled-field loss (config preservation) or hand-edits is worth building, not just documenting. *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-31.** Before any irreversible release step (tag → GitHub Release → PyPI publish), gate on two sweeps. (1) A version number is a claim across four registries that never sync — source (pyproject/__version__), remote git tags, GitHub Releases, the PyPI index — plus the ordering invariant that origin/main must hold the staged commit BEFORE any tag/Release (a GitHub-UI release tags the REMOTE head, so locally-authored guards are unpushed by construction); sweep all four with fresh eyes (uvx by-name answers from uv's cache and can hide a failed attempt — v0.7.0/v0.8.0 were double-claimed by exactly this). (2) Run the suite on EVERY host leg the CI matrix covers — a green on one OS is a guess about the others and the publish cannot be undone; a path/separator assertion is itself a platform claim (assert the wrapper FILE hook.sh|hook.cmd, not the slash). 0.14.0 shipped with 3 Windows cells red on such an assertion. (Consolidates L-08, L-20.) *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-11.** Declare phase dependencies by technical need, not narrative order — and expect a restart-gated exit criterion to split its phase across sessions by design: the session that ships a change to session wiring cannot observe its own next cold start. Record the residue as an explicit REMAINING EXIT CRITERION line in the outputs registry; the next session then closes the phase cold from that line alone, while genuinely independent phases run in the meantime. *(from 2026-06-10-harness-truth-and-release-ritual)*
**L-38.** When an exit criterion is over-specified relative to what is soundly buildable under a constraint (e.g. a no-ML lexical engine cannot decide a SEMANTIC 'alternative' edge, D-018), record an honest AMENDMENT with the reason rather than faking the checkbox - the gameplan's own discipline (D-026 empirical gain-gate, INVARIANT-05) is better served by a truthful scope adjustment than a green box. *(from 2026-06-21-empirical-self-improvement-loop-telemetry-gated-curator-and-loop-gameplan-primitive)*
