**L-42.** release-check's clean_tree counts UNTRACKED files (foreign tool artifacts, regenerable caches) as a dirty tree and blocks the ritual, even though the published artifact builds from origin/main + the tag (which never contain them). **L-31.** Before any irreversible release step (tag → GitHub Release → PyPI publish), gate on two sweeps. (1) A version number is a claim across four registries that never sync — source (pyproject/__version__), remote git tags, GitHub Releases, the PyPI index — plus the ordering invariant that origin/main must hold the staged commit BEFORE any tag/Release (a GitHub-UI release tags the REMOTE head, so locally-authored guards are unpushed by construction); sweep all four with fresh eyes (uvx by-name answers from uv's cache and can hide a failed attempt — v0.7.0/v0.8.0 were double-claimed by exactly this). (2) Run the suite on EVERY host leg the CI matrix covers — a green on one OS is a guess about the others and the publish cannot be undone; a path/separator assertion is itself a platform claim (assert the wrapper FILE hook.sh|hook.cmd, not the slash). 0.14.0 shipped with 3 Windows cells red on such an assertion. (Consolidates L-08, L-20.) *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-39.** A deterministic relevance/gain kill-gate concludes only what its fixtures permit. When the eval corpus is cleanly separable (disjoint topics — as tests/benchmarks RANKER_ENTRIES is), the baseline saturates at the 1.0 recall/nDCG/MRR ceiling, so NO scoring change can show lift: the gate proves "no regression", never "no value", and can only return DISCARD/break-even by construction. To FALSIFY "richer scorer X helps", build the adversarial fixture targeting X's mechanism FIRST — for BM25/Okapi: length-bias confounds, term-frequency skew, near-duplicate topics. Extends L-32 (saturated target ⇒ predict the null) with a fixture-design corollary. Measured instance (2026-06-24-ranker-spike): stdlib BM25 (k1=1.5, b=0.75) tied the raw-overlap baseline at 1.0000 on every extraction+multi_session metric and across a full b-sweep, with guards flat (multi_session precision 0.75, knowledge_updates contradiction 0.0) ⇒ a clean measured DISCARD. Watch-out for any future ranker swap: BM25 length-normalization breaks the stale-vs-current SECONDARY-sort tie that supersession-demotion (analyze.rank_relevant) relies on, so it needs a length-adversarial knowledge_updates fixture before it can be called safe. *(from 2026-06-24-ranker-spike)*
**L-46.** An audit finding names a SYMPTOM plus a HYPOTHESIZED cause, and the hypothesis is falsifiable — measure it before fixing (extends L-39/L-40 from eval-gates to audit-remediation).
