## Project Lessons (most relevant to this phase)

_(5 of 30 shown — ranked by keyword + entity-id overlap with this phase, no ML. The full set is canonical in `docs/LESSONS.md`; the handoff focuses under memory pressure without dropping anything from canonical memory.)_

**L-35.** Cross-model adherence is NOT guaranteed by exposing the MCP tools — a non-Claude model may bypass the tool surface entirely and reach for its own built-ins, the CLI, and raw file edits. Cursor's Composer 2.5 Fast, with the cz_* MCP tools live, made 0 MCP calls across 142 tool calls driving a full gameplan: it recovered from a PATH miss by discovering the portable `uvx … clauderize ops` JSON-batch path on its own (validating that a CLI-reachable fallback for every tracked write [L-05] is load-bearing for cross-MODEL, not just no-MCP sessions), but ALSO hand-edited tracked docs — the frontmatter/graph-corruption anti-pattern the blessed writes exist to prevent. Three takeaways: the CLI/ops fallback earns its place; trust-but-VERIFY a non-Claude model's self-reported close-out against engine state (its plausible summary claimed graph entities a `reindex` showed were never created); and the 'never hand-edit' floor instruction alone does not steer every model — a guard that detects unmodeled-field loss (config preservation) or hand-edits is worth building, not just documenting. *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-43.** A parked feature branch that has fallen behind a newer release reconciles cleanly when the two feature sets are orthogonal in code: dry-run the merge (git merge --no-commit main) to MEASURE the real conflict surface before assuming a heavy rewrite. Here main(1.2.0) auto-merged into the abstract-index branch with a single config-pointer conflict; the union suite went green with zero manual code fixes. Complete parked work by reconciling with the release line FIRST, then finishing the remaining phases against the real target. *(evidence: abstract-index Phase 6/7 resume 2026-06-28: merged main into feat/abstract-index-fast-retrieval, 1 conflict (.clauderizer/config.toml), union suite 711, PR #16 9-cell green)* *(from 2026-06-25-abstract-index-fast-retrieval)*
**L-31.** Before any irreversible release step (tag → GitHub Release → PyPI publish), gate on two sweeps. (1) A version number is a claim across four registries that never sync — source (pyproject/__version__), remote git tags, GitHub Releases, the PyPI index — plus the ordering invariant that origin/main must hold the staged commit BEFORE any tag/Release (a GitHub-UI release tags the REMOTE head, so locally-authored guards are unpushed by construction); sweep all four with fresh eyes (uvx by-name answers from uv's cache and can hide a failed attempt — v0.7.0/v0.8.0 were double-claimed by exactly this). (2) Run the suite on EVERY host leg the CI matrix covers — a green on one OS is a guess about the others and the publish cannot be undone; a path/separator assertion is itself a platform claim (assert the wrapper FILE hook.sh|hook.cmd, not the slash). 0.14.0 shipped with 3 Windows cells red on such an assertion. (Consolidates L-08, L-20.) *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-33.** A subagent's output needs two guards. (1) Its factual file:line CLAIMS are leads to VERIFY at the point of edit, not facts — fan-out exploration locates code fast but doesn't certify it (an exploration map asserted the MCP/CLI parity test lived in test_blessed_surfaces.py; it was test_ops.py, so editing on the map alone would have changed the wrong test). (2) A subagent that RUNS code leaks probe ARTIFACTS into the real repo despite a 'use a throwaway dir' instruction — and an .md left in docs/ is ingested by the graph builder as a phantom entity. Give it an ABSOLUTE artifact dir ('write under /tmp/<unique>/, never the repo; verify pwd before any write') — the explicit instruction measurably worked where the vague one leaked — and after any code-running subagent round, git-status + remove untracked junk (then reindex) BEFORE committing, never a blanket git add -A. (Consolidates L-15, L-30.) *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-34.** The phase that ADDS a field/branch/event is not the phase that OWNS the shared function it must thread through — per-phase TDD covers each phase's new code but misses cross-cutting integration with EXISTING shared paths. When a feature adds a field/branch an existing shared function (config merge, the markdown writer, the graph builder, the hook dispatcher, status_bundle) must also handle, add an explicit integration test AT that seam, OR run an independent post-implementation review pass. Repeatedly load-bearing: independent reviews on this gameplan caught merge_missing dropping host_target, a doctor that false-failed non-claude repos, the hook dispatcher not threading native cross-host event names, and a path-safety asymmetry — none of which the phase's own tests covered. *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
