**L-39.** A deterministic relevance/gain kill-gate concludes only what its fixtures permit. When the eval corpus is cleanly separable (disjoint topics — as tests/benchmarks RANKER_ENTRIES is), the baseline saturates at the 1.0 recall/nDCG/MRR ceiling, so NO scoring change can show lift: the gate proves "no regression", never "no value", and can only return DISCARD/break-even by construction. To FALSIFY "richer scorer X helps", build the adversarial fixture targeting X's mechanism FIRST — for BM25/Okapi: length-bias confounds, term-frequency skew, near-duplicate topics. Extends L-32 (saturated target ⇒ predict the null) with a fixture-design corollary. Measured instance (2026-06-24-ranker-spike): stdlib BM25 (k1=1.5, b=0.75) tied the raw-overlap baseline at 1.0000 on every extraction+multi_session metric and across a full b-sweep, with guards flat (multi_session precision 0.75, knowledge_updates contradiction 0.0) ⇒ a clean measured DISCARD. Watch-out for any future ranker swap: BM25 length-normalization breaks the stale-vs-current SECONDARY-sort tie that supersession-demotion (analyze.rank_relevant) relies on, so it needs a length-adversarial knowledge_updates fixture before it can be called safe. *(from 2026-06-24-ranker-spike)*
**L-32.** Test a borrowed or proposed idea as a falsifiable hypothesis with a pre-named, machine-checkable keep/discard metric, and MEASURE before shipping — a discard is a successful outcome, not a failure; the deliverable is the verdicts plus the survivors (a digest reorder 'worked' by its prefix-stability proxy yet was discarded because the payload was ~222 tokens rendered once per session). Corollary: when the gate's TARGET metric is already SATURATED by a simpler earlier phase, you can PREDICT the zero marginal gain — park the feature by analysis WITHOUT building it (cite the saturated metric + the absent need + the cost), rather than building only to measure the zero the prior phase's number already implies (bitemporal parked because Phase 4 already drove contradiction-rate to its 0.0 floor). (Consolidates L-17, L-27.) *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-40.** When you author BOTH an eval fixture and the detector it scores, a 100%-detection / 0-false-positive result is suspicious by construction — you may be teaching to the test (the confirmation-bias face of L-32/L-39). Give the fixture discriminating power: seed adversarial NEAR-MISSES (a long-but-hollow note; a paper citation that is ALSO in-repo-anchored; a factual "all criteria met" with no intensifier) and run a deliberately NAIVE strawman detector beside the real one. The result is credible only if the principled detector beats the strawman ON the near-misses (a real mechanism), not merely beats a prior no-check baseline (a tautology — anything beats zero). Extends L-39: don't just build the adversarial fixture first — build a strawman the fixture must DEFEAT. Measured: 2026-06-24-critique-bias-hardening, principled 100% det / 0% FP vs naive 20-50% FP; naive wrong on 6/7 near-misses, principled on 0. *(from 2026-06-24-critique-bias-hardening)*
**L-38.** When an exit criterion is over-specified relative to what is soundly buildable under a constraint (e.g. a no-ML lexical engine cannot decide a SEMANTIC 'alternative' edge, D-018), record an honest AMENDMENT with the reason rather than faking the checkbox - the gameplan's own discipline (D-026 empirical gain-gate, INVARIANT-05) is better served by a truthful scope adjustment than a green box. *(from 2026-06-21-empirical-self-improvement-loop-telemetry-gated-curator-and-loop-gameplan-primitive)*
