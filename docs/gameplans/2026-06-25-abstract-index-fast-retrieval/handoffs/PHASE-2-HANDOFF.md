**1.** A dogfood self-check that shells out (preflight's test command) must not depend on the ambient PATH. ## Phase Notes (agent-owned)

**State after Phase 1.** Baseline **651**. `graph/abstract_index.py` exists
(`build`/`load_or_rebuild`, body-free records `{id,title,abstract,anchor,token_set,
content_hash,status,kind}`); NO consumer wired. Phase 2 wires the consumers.

**Key design point — the index is a POINTER, not a body store (D-013).** Records
deliberately omit the full body. `cz_get` resolves the body from CANONICAL markdown
on demand (the index gives kind+anchor; re-parse that ONE corpus file for the entry).
The agent still gets only the one entry (the cost win); the engine's file read is
cheap CPU. Do NOT add bodies to the index — that would duplicate canonical memory.

**Build (verify seams at point of edit — L-33):**
- `analyze.get_entry(id)` — look the id up in `abstract_index.load_or_rebuild(paths)`
  to get its kind/anchor, then re-parse the one corpus file (reuse
  `analyze.parse_entries` for em-dash kinds; the `**L-NN.**` line for lessons) and
  return `{id,title,body,status,anchor,kind}`; `None` if absent.
- `ops.cz_get(id, kind="auto")` — wraps `get_entry`. **Read-only**: `Op(cz_get,
  writes=False)`, no write lock on the read path (L-03), INVARIANT-06 safe. Needs a
  docstring + JSON-serializable defaults (introspection tests).
- `cz_analyze` abstract field — enrich each ranked hit in `analyze.analyze`
  (analyze.py:317-332; the `{id,title,score}` from `rank_relevant` at :118) with the
  entry's `abstract` from the index, so the agent answers from the abstract without a
  `cz_get`. NOTE this threads through `analyze.analyze`, which the **hook handler also
  calls** (`hook/handlers.py:138`, k=3 on every UserPromptSubmit) — the L-34 seam:
  add an integration test there, and keep it read-only/fast (INVARIANT-06).

**Tool-surface parity — update ALL or the suite fails (subagent-verified):**
`tools_list.TOOL_NAMES` (+`cz_get`); `ops.REGISTRY` (+`"cz_get": Op(cz_get,
writes=False)`); tests `test_ops.py::test_registry_is_exactly_the_tool_surface` +
`::test_mcp_registers_the_registry_functions`; `test_ops_introspection.py` (both
tests — docstring + JSON-serializable defaults); `test_skill_discovery.py` parity.
**DEFER** the human-doc tool-count bump (README "N tools", `docs/subsystems/
mcp-server.md` reads count 38→39) to Phase 7 per L-21 / O-04 (don't sweep counts now).

**Exit discipline.** cz_get returns one entry's body for an id from each of the four
corpora; cz_analyze hits carry `abstract`; full parity green; cz_get read-only
