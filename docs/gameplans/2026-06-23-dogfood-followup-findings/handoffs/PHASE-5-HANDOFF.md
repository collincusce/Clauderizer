**L-35.** Cross-model adherence is NOT guaranteed by exposing the MCP tools — a non-Claude model may bypass the tool surface entirely and reach for its own built-ins, the CLI, and raw file edits. Cursor's Composer 2.5 Fast, with the cz_* MCP tools live, made 0 MCP calls across 142 tool calls driving a full gameplan: it recovered from a PATH miss by discovering the portable `uvx … clauderize ops` JSON-batch path on its own (validating that a CLI-reachable fallback for every tracked write [L-05] is load-bearing for cross-MODEL, not just no-MCP sessions), but ALSO hand-edited tracked docs — the frontmatter/graph-corruption anti-pattern the blessed writes exist to prevent. Three takeaways: the CLI/ops fallback earns its place; trust-but-VERIFY a non-Claude model's self-reported close-out against engine state (its plausible summary claimed graph entities a `reindex` showed were never created); and the 'never hand-edit' floor instruction alone does not steer every model — a guard that detects unmodeled-field loss (config preservation) or hand-edits is worth building, not just documenting. *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-26.** Re-distill curated, append-only memory behind a COVERAGE GATE, never by taste: auto-derive a query from each lesson's OWN distinctive tokens (no hand-picked probes) and require that after consolidation the lesson's synthesis still surfaces in the ranker top-k; merge only the tightest clusters, mark sources obsolete with a "consolidated into L-NN" pointer (never delete — INVARIANT-03), and prove coverage both BEFORE apply (simulated) and AFTER apply (on the live file, with the marker stripped so the check cannot cheat). Instance: docs/LESSONS.md 21 → 16 active at 9/9 post-apply coverage, rollup −20%. *(from 2026-06-20-empirical-memory-gains)*
**L-33.** A subagent's output needs two guards. (1) Its factual file:line CLAIMS are leads to VERIFY at the point of edit, not facts — fan-out exploration locates code fast but doesn't certify it (an exploration map asserted the MCP/CLI parity test lived in test_blessed_surfaces.py; it was test_ops.py, so editing on the map alone would have changed the wrong test). (2) A subagent that RUNS code leaks probe ARTIFACTS into the real repo despite a 'use a throwaway dir' instruction — and an .md left in docs/ is ingested by the graph builder as a phantom entity. Give it an ABSOLUTE artifact dir ('write under /tmp/<unique>/, never the repo; verify pwd before any write') — the explicit instruction measurably worked where the vague one leaked — and after any code-running subagent round, git-status + remove untracked junk (then reindex) BEFORE committing, never a blanket git add -A. (Consolidates L-15, L-30.) *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
**L-25.** A health check / guard must verify CAPABILITY, not presence - a green check on a non-launchable setup is worse than none. Prove every probe at the granularity it reports, in BOTH directions (it must fire on the failure it exists for and stay green on health, per matrix cell / per argument), and prefer in-band identity (the output must claim who and what version ran) over exit codes: locally-sound guards compose into false green (a wrapper that always exits 0 defeats a spawn probe that reads exit 0 as launchable). The probe must traverse the consumer's exact execution leg, shell quirks included. (Consolidates L-02, L-09, L-10.) *(from 2026-06-20-empirical-memory-gains)*
**L-29.** Isolate destructive/irreversible operations from the real repo BEFORE running them, and prove the isolation — never assume it. A destructive CLI smoke (e.g. `clauderize uninstall`, which rmtree's `.clauderizer/` and git-deletes tracked wiring) run via `cd $(mktemp -d)` silently executed against the dogfood repo because, inside `wsl.exe -d ubuntu bash -lc '…'` invoked by the Git-Bash Bash tool, `$(…)`/`$VAR`/`$$` expand in the OUTER Windows shell (empty / a Git-Bash path that doesn't exist in WSL), so `cd ""` stayed put. Reliable isolation: Python `tempfile.mkdtemp()` in a `.venv/bin/python` script (runs in WSL), or the Write tool to a `\\wsl.localhost\…\tmp\` path, or literal paths — never shell `$()`/`$VAR`. ALWAYS guard before the destructive step: `case "$PWD" in *<repo-name>) exit 9;; esac` (it fired and prevented a repeat). And prefer the guaranteed-isolated pytest `tmp_path` tests that usually already cover the path — a manual smoke of a destructive command adds real risk for near-zero coverage. Recovery here was clean only because uninstall is non-destructive-by-design (tracked files, `git checkout` restores; `docs/` preserved). *(from 2026-06-21-cross-host-cross-model-clauderizer-universal-agents-md-mcp-substrate)*
